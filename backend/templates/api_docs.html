üí° Usage Examples

1. Memulai Optimasi Rute

curl -X POST http://localhost:5000/api/optimize \ -H "Content-Type: application/json" \ -d '{ "preferences": { "budget": 150000, "max_places": 5, "preferred_categories": ["Budaya", "Alam"], "must_visit": [1, 2], "start_time": "09:00", "end_time": "18:00" } }'
Response:
{ "status": "success", "data": { "route": [1, 2, 3, 5, 8], "fitness": 85.5, "schedule": [...], "total_cost": 145000 }, "message": "Rute berhasil dioptimasi" }

3. API Gabungan - Next Place + Reoptimize

curl -X POST http://localhost:5000/api/next-and-reoptimize
Response:
{ "status": "success", "data": { "success": true, "step1_next_place": { "success": true, "current_position": 2, "current_place": { "id": 2, "name": "Candi Prambanan", "category": "Budaya" } }, "step2_reoptimize": { "success": true, "dynamic_data": { "weather": "Cerah", "crowdedness_factor": 1.5 }, "updated_route": ["Candi Borobudur", "Candi Prambanan", ...] }, "summary": { "moved_to": "Candi Prambanan", "new_position": 2, "route_updated": true, "weather_condition": "Cerah" }, "server_time": "15:45", "timestamp": "2025-05-25T15:45:30.123456" }, "message": "Berhasil pindah ke Candi Prambanan dan rute telah dioptimasi ulang" }

4. Workflow Perjalanan Lengkap

1 Optimasi Rute: POST /api/optimize
2 Cek Status: GET /api/status
3 Lanjut + Reoptimasi Otomatis: POST /api/next-and-reoptimize
4 Manual - Lanjut ke Tempat Berikutnya: POST /api/next-place
5 Manual - Reoptimasi (jika perlu): POST /api/reoptimize
6 Akhiri Perjalanan: POST /api/end-journey

5. Response Format Standar

{ "status": "success|error", "data": { // Response data }, "message": "Descriptive message" }
‚ö†Ô∏è Catatan: Pastikan untuk melakukan optimasi rute terlebih dahulu sebelum menggunakan endpoint lainnya. Gunakan /api/next-and-reoptimize untuk kemudahan otomatis.
